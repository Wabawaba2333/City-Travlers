# 城镇探索者 City-Travlers（Pygame 2D 像素游戏）

> 请永远用中文回答我。  
> **最后更新**：2025-10-18

## 1. 概览
- **引擎**：Pygame 2（SDL2）
- **目标平台**：桌面端（Windows 为主）
- **逻辑分辨率**：1280×720（窗口态）
- **帧率**：FPS = 60
- **入口**：`python game.py`  
  启动前会自动对 **TSX/TMX** 做路径与坐标预处理，然后进入游戏。

## 2. 运行期显示行为（必须遵守）
- **启动即进入**：全屏 **无标题栏**、**无黑边**、**不改变系统分辨率**（伪全屏或 SCALED）。
- **按一次 `ESC`**：切到 **1280×720 有标题栏窗口**。
- **再按一次 `ESC`**：回到 **全屏无标题栏**。  
  以上两态 **循环切换**。
- **窗口标题栏“最大化按钮”** 的效果 **= 按一次 ESC**（窗口态最大化 ⇒ 全屏；全屏态无标题栏按钮）。
- 切换不会影响系统中**其他应用窗口**的大小或分辨率；切换时**不销毁游戏状态**（玩家位置/相机/关卡对象应保持）。

## 5. 传送逻辑（新增）

传送功能里每张地图是独立加载的，但在逻辑上相互相邻，玩家在到达当前地图的某个边界触发传送后，应被移动到目标地图对应边界的合理位置。传送的核心思路是：当玩家从一个地图的边缘区域进入传送状态时，根据对应地图的传送区域内的相对位置比例计算落点，而传送方向垂直轴的坐标通过映射比例计算，传送方向的坐标轴则严格固定为指定坐标。例如当玩家从左街区往右走进入钟楼广场时，除固定轴外的另一条轴采用钟楼广场传送区域（x：1–45，y：1955–2212）的相对位置计算。要求从地图A传送到地图B后，其中一条坐标轴的值必须严格等于给定的固定坐标，完全一致。每张地图之间的传送冷却时间应独立，不应共用三秒计时。并且传送后，如果玩家没有主动按下移动键，角色将永远不会因仍处在传送区域内再次被传送。

传送逻辑执行分两阶段。第一阶段在源地图触发时，计算玩家在源传送区域内的相对位置，保存相对比例、固定轴与源配置。第二阶段在切换至目标地图后，查找目标地图中与固定轴匹配的回程传送区域，并使用相对比例计算另一条轴的坐标，同时用源配置中定义的固定坐标作为固定轴坐标，从而得到精确落点。以左街区传送至钟楼广场为例：源区域为 (3733, 742, 3789, 1015)，玩家 y=913，对应相对位置 62.6%，固定坐标为 x=30。切换地图后查找钟楼广场中 fixed_axis='x' 的对应区域 (1,1955,45,2212)，计算 y=1955 + 0.626×(2212-1955)=2116，因此最终落点为 (30, 2116)。
